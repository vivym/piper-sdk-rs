# 报告修订说明（v1.0 → v2.0）

**修订日期**: 2025-01-29
**修订原因**: 技术评审反馈

---

## 关键修正

### 1. ⚠️ CRITICAL: 修正了 `forward()` vs `mj_inverse()` 的技术错误

**原错误（v1.0）**:
```rust
// ❌ 错误建议：使用 forward() 计算完整逆动力学
fn compute_inverse_dynamics(q, qvel, qacc) {
    self.data.qacc_mut()[0..6].copy_from_slice(qacc);
    self.data.forward();  // 错误！
    return self.data.qfrc_bias();  // 不包含惯性项
}
```

**修正后（v2.0）**:
```rust
// ✅ 正确做法：必须使用 mj_inverse()
fn compute_inverse_dynamics(q, qvel, qacc) {
    self.data.qacc_mut()[0..6].copy_from_slice(qacc);
    unsafe {
        mujoco_rs::sys::mj_inverse(self.model.ffi(), self.data.ffi());
    }
    return self.data.qfrc_inverse();  // 包含完整逆动力学
}
```

**技术说明**:
- `qfrc_bias` 的定义就是：重力 + 科里奥利力 + 离心力 + 弹簧 + 阻尼
- `qfrc_bias` **永远不包含**惯性项 `M(q)·q̈`
- 完整逆动力学**必须**使用 `mj_inverse()`，结果在 `qfrc_inverse` 中

---

### 2. ✅ 澄清了"重力补偿" vs "逆动力学"的概念

**原观点（v1.0）**:
- 当前实现是"错误的"，因为它没有使用实际速度
- 应该强制修改现有实现

**修正后（v2.0）**:
- 当前实现是**语义正确**的纯重力补偿
- 参考实现是部分逆动力学（适用于不同场景）
- 应该提供**多种模式供用户选择**

---

### 3. ✅ 重新评估了 `mj_jac` FFI 的必要性

**原观点（v1.0）**:
- 使用 `mj_jac` 是"不推荐"的底层实现
- 应该替换为 `jac_site` 高级 API

**修正后（v2.0）**:
- `mj_jac` 在负载补偿场景下是**必要之恶**
- 它是唯一能计算**任意点**（动态质心）Jacobian 的方法
- `jac_site` 只能计算固定 Site，无法满足动态负载需求
- 应该保留 `mj_jac` 但封装为安全的私有方法

---

### 4. ✅ 添加了摩擦力/阻尼的说明

**新增内容**:
- 部分逆动力学会自动补偿 XML 中定义的关节阻尼
- `qfrc_bias` 包含：重力 + 科里奥利力 + 离心力 + 弹簧 + **阻尼 + 摩擦**
- 这是高精度跟踪的重要加分项
