## 🚨 关键修正：循环漂移问题（v3.2）

### 问题分析

**v3.1 设计**：
```rust
let sleep_duration = Duration::from_secs_f64(1.0 / self.config.control_rate); // 5ms

while start.elapsed() < timeout {
    self.command_joints(target, None)?;  // 假设耗时 0.5ms
    // ...
    spin_sleep::sleep(sleep_duration);  // 固定睡眠 5ms
}
```

**问题**：
- ❌ 实际循环周期 = `T_cmd + sleep_duration` = 0.5ms + 5ms = 5.5ms
- ❌ 实际控制频率 = 1000ms / 5.5ms ≈ **181Hz**（而非预期的 200Hz）
- ❌ 如果 `T_cmd` 有波动，控制频率也会随之波动

**根本原因**：
1. **累积漂移**：每周期多出的 `T_cmd` 时间会累积
2. **频率不稳定**：依赖 CAN 发送耗时，导致控制频率波动

---

### 修正方案：绝对时间锚点

**v3.2 修正**：
```rust
impl MitController {
    pub fn move_to_position(
        &mut self,
        target: [Rad; 6],
        threshold: Rad,
        timeout: Duration,
    ) -> Result<bool, ControlError> {
        const MAX_TOLERANCE: u32 = 5;
        let mut error_count = 0;

        let start = Instant::now();

        // ⚠️ v3.2: 使用绝对时间锚点
        let period = Duration::from_secs_f64(1.0 / self.config.control_rate);  // 5ms
        let mut next_tick = Instant::now();

        while start.elapsed() < timeout {
            // 1. 设定下一个锚点（绝对时间）
            next_tick += period;

            // 2. 发送命令 & 检查状态（耗时操作）
            match self.command_joints(target, None) {
                Ok(_) => error_count = 0,
                Err(e) => {
                    error_count += 1;
                    if error_count > MAX_TOLERANCE {
                        error!("Consecutive CAN failures: {}", error_count);
                        return Err(ControlError::ConsecutiveFailures {
                            count: error_count,
                            last_error: Box::new(e),
                        });
                    }
                    warn!("Transient CAN error: {}, skipping frame", e);
                    // 跳过本帧，继续循环
                    continue;
                }
            }

            // 3. 检查是否到达
            let current = self.observer.joint_positions();
            let reached = current.iter()
                .zip(target.iter())
                .all(|(c, t)| (c - t).abs() < threshold);

            if reached {
                return Ok(true);
            }

            // 4. ⚠️ v3.2: 睡眠到下一个锚点（自动扣除耗时操作的时间）
            let now = Instant::now();
            if next_tick > now {
                // 还有剩余时间，睡眠到下一个锚点
                spin_sleep::sleep(next_tick - now);
            } else {
                // ⚠️ 任务超时（Overrun）：耗时操作超过了预期周期
                warn!(
                    "Control loop overrun: operation took {:?}, \
                     but next tick was {:?} from now. \
                     Skipping sleep to catch up.",
                    now.elapsed_since(next_tick - period),
                    next_tick.duration_since(now)
                );
                // 追赶锚点：不睡眠，直接进入下一帧
                next_tick = now;  // 重置锚点
            }
        }

        Ok(false)
    }
}
```

**收益**：
- ✅ 无论 `command_joints` 耗时多少（只要 < period），控制频率都能锁定在 200Hz
- ✅ 消除累积漂移，控制频率精确稳定
- ✅ 自动处理超时（Overrun）情况
- ✅ 支持任务耗时波动（CAN 发送有 jitter）

#### ⚠️ 关键优化 2：循环锚点修正（v3.2 新增）⭐

**完整实现代码**：

```rust
impl MitController {
    /// 阻塞式运动到目标位置（带容错性 + 循环锚点）
    ///
    /// # 容错性
    ///
    /// 控制循环具有**容错性**：允许偶尔的 CAN 通信错误（最多连续 5 帧）。
    ///
    /// # 控制频率精度 ⚠️ v3.2
    ///
    /// 使用**绝对时间锚点**机制，消除累积漂移，保证精确的 200Hz 控制频率：
    /// - 无论 CAN 通信耗时多少（只要 < 5ms），控制频率都锁定在 200Hz
    /// - 自动处理任务超时（Overrun）情况
    /// - 支持任务耗时波动（jitter）
    ///
    /// # 参数
    ///
    /// - `target`: 目标关节位置（弧度）
    /// - `threshold`: 到达阈值（弧度）
    /// - `timeout`: 超时时间
    ///
    /// # 返回
    ///
    /// - `Ok(true)`: 到达目标
    /// - `Ok(false)`: 超时未到达
    /// - `Err(ControlError::ConsecutiveFailures)`: 连续错误超过阈值
    ///
    /// # 示例
    ///
    /// ```rust
    /// let mut controller = MitController::new(piper, config)?;
    ///
    /// controller.move_to_position(
    ///     [Rad(0.5), Rad(0.7), Rad(-0.4), Rad(0.2), Rad(0.3), Rad(0.5)],
    ///     Rad(0.01),
    ///     Duration::from_secs(5.0),
    /// )?;
    /// ```
    pub fn move_to_position(
        &mut self,
        target: [Rad; 6],
        threshold: Rad,
        timeout: Duration,
    ) -> Result<bool, ControlError> {
        const MAX_TOLERANCE: u32 = 5;  // 允许连续丢 5 帧（25ms @ 200Hz）
        let mut error_count = 0;

        let start = Instant::now();

        // ⚠️ v3.2: 使用绝对时间锚点，消除循环漂移
        let period = Duration::from_secs_f64(1.0 / self.config.control_rate);  // 5ms @ 200Hz
        let mut next_tick = Instant::now();

        while start.elapsed() < timeout {
            // 1. 设定下一个锚点（绝对时间）
            next_tick += period;

            // 2. 发送命令 & 检查状态（耗时操作）
            match self.command_joints(target, None) {
                Ok(_) => {
                    error_count = 0;  // 重置计数器
                }
                Err(e) => {
                    error_count += 1;
                    if error_count > MAX_TOLERANCE {
                        error!(
                            "Consecutive CAN failures ({}): {:?}. Aborting motion.",
                            error_count, e
                        );
                        return Err(ControlError::ConsecutiveFailures {
                            count: error_count,
                            last_error: Box::new(e),
                        });
                    }
                    warn!(
                        "Transient CAN error ({}): {:?}, skipping frame. \
                         This is acceptable as long as errors don't occur consecutively.",
                        error_count, e
                    );
                    // 跳过本帧，继续循环
                    continue;
                }
            }

            // 3. 检查是否到达
            let current = self.observer.joint_positions();
            let reached = current.iter()
                .zip(target.iter())
                .all(|(c, t)| (c - t).abs() < threshold);

            if reached {
                return Ok(true);
            }

            // 4. ⚠️ v3.2: 睡眠到下一个锚点（自动扣除耗时操作的时间）
            let now = Instant::now();
            if next_tick > now {
                // 还有剩余时间，睡眠到下一个锚点
                spin_sleep::sleep(next_tick - now);
            } else {
                // ⚠️ 任务超时（Overrun）：耗时操作超过了预期周期
                warn!(
                    "Control loop overrun: operation took {:?}, \
                     but next tick was {:?} from now (expected {:?}). \
                     Skipping sleep to catch up.",
                    now.duration_since(next_tick - period),
                    next_tick.duration_since(now),
                    period
                );
                // 追赶锚点：不睡眠，直接进入下一帧
                // 重置锚点到当前时间，避免后续累积延迟
                next_tick = now;
            }
        }

        Ok(false)
    }
}
```

**技术原理**：

```
v3.1 (有漂移):
Frame 0:  0ms (start) → cmd 0.5ms → sleep 5ms → 5.5ms (实际)
Frame 1:  5.5ms → cmd 0.5ms → sleep 5ms → 11.0ms (实际)
Frame 2:  11.0ms → cmd 0.5ms → sleep 5ms → 16.5ms (实际)
→ 频率: 1000ms / 5.5ms ≈ 181Hz ❌

v3.2 (锚点修正):
Frame 0:  0ms → cmd 0.5ms → sleep 4.5ms → 5ms (锚点)
Frame 1:  5ms → cmd 0.5ms → sleep 4.5ms → 10ms (锚点)
Frame 2:  10ms → cmd 0.5ms → sleep 4.5ms → 15ms (锚点)
→ 频率: 1000ms / 5ms = 200Hz ✅ (稳定)
```

**关键收益**：
1. ✅ **精确频率**：无论 CAN 耗时多少（只要 < 5ms），频率都锁定在 200Hz
2. ✅ **消除漂移**：不累积时间误差
3. ✅ **处理超时**：自动检测并处理 overrun 情况
4. ✅ **稳定性**：即使 CAN 通信有 jitter，控制频率仍然稳定

```rust
// ✅ v3.2: 锚点机制（消除漂移）
let mut next_tick = Instant::now();

// 执行耗时操作...
